FindAccess * Auffinden von Speicherzugriffen

Dieses Programm lädt die angegebene Datei und durchsucht sie nach Zugriffen auf eine angegebene Adresse. Es findet alle absoluten und relativen 8-bit, 16-bit und 32-bit Referenzen. Es wurde geschrieben, um CoreDump Dateien zu untersuchen. Es ist ähnlich zum Freezer Kommando FA, disassembliert aber die gefundenen Zugriffe nicht, sondern zeigt sie als HexDump.

Installation

FindAccess ist nur im DEV, nicht im USR Archiv enthalten.
Kopiere das Programm nach C: oder einem ähnlichen Platz.

Benutzung

Empfehlenswert ist die Benutzung durch folgendes Alias (z.B. permanent durch Hinzufügen zu S:Shell-Startup), welches das letzte von WHDLoad geschriebene Memory Dump (den Pfad evtl. anpassen) durchsucht:

alias fa FindAccess C:.whdl_memory

damit kann man in einer Shell eingeben:

fa $15000

und bekommt als Ausgabe:

FindAccess 1.2 (17-Jan-99 14:45:18) by Bert Jahn
loading file Workbench:CE/.whdl_memory
scanning accesses to $15000 (86016) file: $0-$80000
Relative Word at $ 14240 -> 06C00000 07C00000 0DC0 00000CC0 000004C0

Optionen

FILE/A,ADDRESS/A,ORG
FILE/A Name der Datei die durchsucht werden soll
ADDRESS/A   die Adresse nach dessen Zugriff gesucht werden soll, es können einfache Ausdrücke wie "$5000" = "20480" = " -$Ff +-33+ 19512+$4e8" (Prefix "$" für hexadezimale Zahlen, unterstützte Operatoren sind "+" and "-", unär und binär) verwendet werden
ORG die logische Startadresse der Datei, wenn nicht angegeben, wird als $ 0 angenommen, es können dieselben Ausdrücke wie bei ADDRESS/A verwendet werden
