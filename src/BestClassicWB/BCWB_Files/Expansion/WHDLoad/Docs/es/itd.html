ITD * Imagen A Disco

Herramienta para escribir imágenes de disco nuevamente a un dispositivo de disco.

Instalación

ITD solamente se encuentra en el archivo DEV archive, no en el archivo USR.
Cópielo a C: o una ubicación similar

Opciones

File/A,Device,Format/S,BS=BlockSize/N,ASync/S,ForceOverwriteLargeDevice/S
File/A nombre del archivo que contiene la imagen de disco a escribir al dispositivo
Device el dispositivo al cual ITD escribirá la imagen
por defecto "DF0:"
ejemplo "RAD:"
Format/S   si se especifica el disco también será formateado durante la escritura, para la mayoría de los dispositivos esto no hace ninguna diferencia
BlockSize/N la cantidad de información leída y escrita durante cada ciclo, el valor por defecto es el tamaño de una pista (BlocksPerTrack*SizeBlock)
si se opera en dispositivos grandes se puede mejorar el rendimiento utilizando un tamaño más grande de bloque pero cuidado porque algunos dispositivos no funcionan correctamente con tamaños de bloques superiores a 130,560 ($1fe00) o aún 65,024 ($fe00)
ASync/S si se activa ITD escribirá en forma asíncrona al dispositivo destino
no lo utilice si el archivo de origen y destino están ubicados en el mismo dispositivo físico, porque enlentecerá la operación
si al menos uno de los dispositivos (origen o destino) soporta DMA esto mejorará significativamente el rendimiento de la operación de copia
ForceOverwriteLargeDevice/S Normalmente ITD no leerá de archivos o escribirá a dispositivos más grandes que 2,000,000 bytes. Utilizando este conmutador ITD operará en archivos y dispositivos de cualquier tamaño.

Código de retorno

0 si es exitoso, de otra forma 20.

Descripción técnica

ITD puede escribir a cualquier dispositivo de pistas de disco (trackdisk). Para evitar problemas (por ej. corrupción del disco duro) no escribe a dispositivos mas grandes que 2,000,000 bytes excepto si se utiliza la opción ForceOverwriteLargeDevice/S. Si la operación de escritura excede el límite de 4 GiB se utilizarán comandos de TrackDisk64.
NO se realizará una verificación.
Si la imagen es mas pequeña que el dispositivo Ud. vera un mensaje de advertencia, ITD no cambiara las pistas excedentes en este caso pero redondeará hacia arriba el tamaño hasta un bloque completo y limpiará el espacio excedente. Si la imagen es más grande Ud. también vera un mensaje e ITD escribirá tanta información como quepa en el disco.
