ITD * Image To Disk

C'est un utilitaire pour copier un fichier image sur une disquette.

Installation

ITD se trouve seulement dans l'archive DEV, non dans l'archive USR.
Copiez-le dans le C: ou dans un emplacement similaire.

Options

File/A,Device,Format/S,BS=BlockSize/N,ASync/S,ForceOverwriteLargeDevice/S
File/A nom du fichier qui contient l'image du disque à écrire sur le périphérique
Device le nom du device que ITD écrira l'image
defaut "DF0:"
exemple "RAD:"
Format/S   si activé, le disque sera formatté pendant l'écriture, pour la plupart des périphériques cela ne fait aucune différence.
BlockSize/N représente le nombre de données lues et écrites durant chaque cycle, la taille par défaut d'une piste est: (BlocksPerTrack*SizeBlock)
sur des périphériques de taille plus importante, l'utilisattion d'un bloc de donnée plus large peut améliorer les performances. Mais attention, certains ne fonctionnent plus correctement si la taille des blocs est supérieure à 130,560 ($1fe00) ou même 65,024 ($fe00)
ASync/S Si l'option activée, ITD écrira les données de façon asynchrone vers le périphérique de destination. Ne l'utilisez pas si le fichier source et le périphérique de destination sont situés sur le même périphérique physique, car l'opération sera ralentie. Par contre, si au moins la source ou le périphérique de destination supporte les canaux DMA, les performances de la copie seront grandement améliorées.
ForceOverwriteLargeDevice/S Normalement, ITD ne lit pas des fichiers ou n'écrit pas sur des périphériques qui sont supérieurs à 2,000,000 octets. Avec cette option, les fichiers et périphériques peuvent avoir n'importe quel taille.

Code de retour

0 si réussite, sinon 20.

Généralité

ITD peut écrire sur n'importe quel trackdisk device. Pour éviter certains problèmes (par ex. altération de données sur votre disque dur), il ne peut inscrire sur des devices plus de 2,000,000 octets sauf avec l'option ForceOverwriteLargeDevice/S. Si l'opération dépasse la barrière des 4GB, il utilisera la commande TrackDisk64.
Il n'y aura pas d'opération de vérification du disque.
Si l'image est plus petite que le périphérique, vous receverez un message d'attention. ITD ne touchera pas aux pistes restantes dans ce cas mais arrondira la taille au chiffre supérieur pour obtenir un bloc complet et effacera l'espace restant de la piste. Si l'image est plus grande, vous receverez aussi un message et ITD écrira seulement les données qui peuvent être inscrites sur le périphérique.
