Accesso alla rete

WHDLoad può utlizzare la rete per controllare se esiste una nuova versione sia dello Slave che di WHDLoad stesso. Le operazioni di rete avvengono tutte tramite il protocollo HTTP. L'opzione globale Proxy/S può essere utilizzata per accedere alla rete tramite un proxy, anche autenticandosi con username e password nel caso sia necessario. WHDLoad supporta solo l'autenticazione semplice.
Se si vuole escludere ogni tipo di accesso di rete è sufficiente specificare l'opzione globale NoNetwork/S.

Controllo nuova versione di WHDLoad e Slave

Accedendo al sito web di WHDLoad verrà verificata l'eventuale presenza di una nuova versione dello Slave o di WHDLoad.
L'accesso viene effettuato solo se la splash window è aperta visto che l'informazione è visualizzata lì.

Il numero della versione di WHDLoad verrà inviata e comparata con la versione presente sul sito e, nel caso una versione più recente sia disponibile, un pulsante 'Update WHDLoad' verrà aggiunto alla splash window. Se avete installato openurl.library premendo il pulsante si aprirà il browser web sul sito di WHDLoad, se la libreria non è installata l'indirizzo della home page verrà semplicemente visualizzato. Se l'opzione Expert/S è stata selezionata, verrete informati anche sul rilascio di nuove versioni beta di WHDLoad.

Per verificare la presenza di uno Slave aggiornato, il suo SHA-256 verrà invitiato e comparato con quello presente sul sito di WHDLoad, dove c'è un database di tutti gli Slave. Se un nuovo Install o Slave è presente, la splash window mostrerà la data di rilascio della versione più recente, aggiungendo un pulsante per l'aggiornamento. Se avete installato openurl.library premendo il pulsante si aprirà il browser web sulla pagina dello specifico Install sul sito WHDLoad, altrimenti il suo indirizzo verrà semplicemente visualizzato.

Se una nuova versione di uno Slave o di WHDLoad sono disponibili, un nuovo pulsante 'Start' verrà aggiunto alla splash window, e questa non verrà chiusa automaticamente per evidenziare la possibilità dell'aggiornamento. La finestra potrà essere chiusa normalmente con il mouse, la barra spazio, Return o Esc.

Se avete installato openurl.library ed il pulsante 'Update' viene premuto, WHDLoad eseguirà il comando ExecuteCleanup/K per riattivare lo stack di rete, di solito disabilitato tramite il comando ExecuteStartup/K. L'operazione può fallire se non c'è memoria libera sufficiente, nel caso Preload/S ne abbia occupata la maggior parte. Questo significa che in situazioni con poca memoria libera il pulsante 'Update' potrebbe non funzionare correttamente.
