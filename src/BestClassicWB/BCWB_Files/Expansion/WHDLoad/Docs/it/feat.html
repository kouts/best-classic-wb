Caratteristiche di WHDLoad

  • avviabile da CLI o da Workbench
  • memoria protetta utilizzando la MMU (se disponibile)
  • ritorno al OS assolutamente pulito (dipende dallo Slave)
  • possibilità di utilizzo di "immagini disco" e/o normali file
  • possibilità di scrivere i file in un'altra parte del file system, per tenere i salvataggi separati
  • dimensione variabile della memoria Chip per il programma installato da $2000 fino a $200000
  • dimensione della memoria Fast illimitata per il programma installato
  • allocazione intelligente della memoria Chip, parzialmente assoluta e parzialmente indipendente
  • cache dei file e delle immagini disco all'avvio e dinamicamente mentre sono in esecuzione (LRU)
  • decompressione automatica di file e immagini disco tramite XPK
  • decompressione automatica dei file se lo Slave lo supporta (RNC/IMP/TPWM/CRM)
  • emulazione di MOVE SR,ea dallo User-Mode (modalità utente) su 68010+
  • emulazione delle istruzioni integer non supportate dal 68060
  • supporto avanzato per il debugging, (memory dump, register dump, logging dell'accesso ai file, logging dell'accesso ai registri Custom)
  • a prova di Enforcer/Cyberguard e VMM se la opzione NoVBRMove/S non viene utilizzata
