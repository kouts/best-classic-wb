ITD * Image To Disk

Tool per scrivere immagini disco in un floppy.

Installazione

ITD è contenuto solo nell'archivio DEV, non in quello USR.
Copialo in C: o in un posto simile

Opzioni

File/A,Device,Format/S,BS=BlockSize/N,ASync/S,ForceOverwriteLargeDevice/S
File/A nome del file che contiene l'immagine del disco da scrivere sul device
Device il device in cui ITD dovrà scrivere l'immagine
default "DF0:"
esempio "RAD:"
Format/S se abilitato il disco verrà anche formattato nella scrittura, per la maggior parte dei device questa cosa non fa differenza
BlockSize/N il quantitativo di dati letti e scritti durante ogni ciclo, di default è la dimensione di una track (BlocksPerTrack*SizeBlock)
quando si lavora su device grandi, usare blocchi più grandi può incrementare le performance, ma bisogna tenere in considerazione che certi device non funzionano correttamente con dimensione dei blocchi superiore a 130,560 ($1fe00) o anche 65,024 ($fe00)
ASync/S se abilitato ITD scriverà in maniera asincrona nel device di destinazione
non va usato se il file sorgente e il device di destinazione sono sullo stesso device fisico, perchè rallenterebbe le operazioni
se almeno uno tra file sorgente e device di destinazione supporta il DMA ci saranno vantaggi significativi in termini di performance dell'operazione di copia
ForceOverwriteLargeDevice/S Di solito ITD non leggerà o scriverà su device che sono più grandi di 2,000,000 byte Utilizzando questa opzione ITD funzionerà su file e device di ogni dimensione.

Return code (codice di ritorno)

Impostato a 0 se è andato tutto bene, altrimenti a 20

Background

ITD può scrivere in ogni device di tipo trackdisk. Per evitare problemi (ad esempio corrompere i dati nel tuo harddisk) per il momento non scriverà in device più grandi di 2.000.000 di byte eccetto quando viene attivata l'opzione ForceOverwriteLargeDevice/S. Se l'operazione di scrittura eccede i 4 GiB utilizzerà comandi TrackDisk64.
NON ci sarà nessuna verifica sui dati scritti.
Se l'immagine è più piccola del device ne verrai avvisato. In questo caso ITD non modificherà le tracce rimanenti, ma arrotonderà la dimensione fino ad un blocco pieno e azzererà lo spazio rimanente. Anche nel caso in cui l'immagine fosse più grande verrai avvertito, e ITD scriverà fino alla fine dello spazio disponibile sul device.
