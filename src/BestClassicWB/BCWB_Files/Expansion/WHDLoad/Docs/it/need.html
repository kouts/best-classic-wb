Requisiti di sistema

... per avviare i programmi installati:

  • CPU 68000 (è raccomandata una CPU 68010+ perché qualche installatore richiede il VBR per il 'quit by a key' (esci con un tasto))
  • Kickstart 2.0 (versione 37)
  • un minimo di 1.0 MB RAM (a volte di più, dipende dal programma installato)
Emulazione Kickstart

Qualche installatore utilizza quello che viene chiamato kickemu per ottenere un ambiente AmigaOS completo per il programma installato. Per il kickemu serve l'immagine di un kickstart. Questa immagine può essere estratta da un vero Amiga con programmi quali GrabKick o simili che possono essere trovati su Aminet (controllate che il setup per i vostri dischi (MaxTransfer, Mask) sia corretto, altrimenti l'immagine creata potrebbe essere corrotta). La versione supportata dell'immagine dipende dal programma installato. Dovresti poter trovare qualche informazione riguardo alla versione da utilizzare nel file ReadMe allegato all'installatore. L'immagine per le versioni 1.2 e 1.3 del kickstart può essere ottenuta solo da un A500, la versione 3.1 può essere estratta da un A500/A600/A2000,A1200 o un A4000 che naturalmente deve avere la Kick3.1 ROM. Se l'immagine non può essere trovata da WHDLoad oppure è la versione sbagliata o alterata in qualche modo, WHDLoad terminerà l'esecuzione con un apposito messaggio d'errore. In aggiunta a questo, un file di rilocazione con l'estensione .RTB associato all'immagine del kickstart deve essere presente. Questo file di rilocazione può essere trovato su Aminet nell'archivio util/boot/skick346.lha. Entrambi i file, l'immagine del kickstart e il file di rilocazione, devono trovarsi nella directory Devs:Kickstarts/ ed essere nominati nel modo corretto. A partire dalla version 17.0 di WHDLoad i files possono anche essere presenti nella directory WHDCOMMON:. La seguente tabella indica i nomi dei file, dipendenti dalla versione del kickstart:

A partire dalla versione 16.5 di WHDLoad possono essere utilizzate anche le immagini del kickstart presenti nel CD Amiga Forever di Cloanto. Le immagini devono essere rinominate seguendo la tabella qui in basso. In aggiunta rom.key deve essere copiata in Devs:Kickstarts/ o in WHDCOMMON:. Alcuni vecchi installatori hanno bisogno dell'immagine kick40068.A1200, questi programmi non funzioneranno con le ROM Cloanto.
Amiga Forever 2008 include il kick40068.A1200 e le immagini saranno automaticamente copiate al posto giusto.
Anche Amiga Forever 2011 contiene le rom kick33180.A500 e kick40063.A600. Queste rom richedono WHDLoad versione 17.0 per funzionare.

pacchetti opzionali:

  • Kickstart 3.1 (versione 40), sistema alcuni problemi delle versioni precedenti, aggiunge anche nuove caratteristiche che sono utilizzate da WHDLoad, se disponibili, come MemoryPools che aumenta le prestazioni se il programma installato utilizza parecchi file.
  • per utilizzare file di dati compressi con XPK, le librerie XPK devono essere installate, è possibile scaricarle su aminet. Tutti i file di dati possono essere compressi utilizzando XPK, HWDLoad li decomprimerà al momento del caricamento, ma la decompressione richiede l'utilizzo di più memoria e quindi deve essere utilizzata con cautela su sistemi con poca memoria libera. I file scritti da WHDLoad non sono mai compressi.
  • per la selezione automatica della lingua utilizzata in alcuni programmi viene utilizzata la lowlevel.library, essa è parte del Workbench 3.1 ma può essere installata su versioni precedenti dell'OS, se lowlevel.library non è presente gli installatori utilizzerano la lingua inglese
  • se la reqtools.library è installata WHDLoad la utilizzerà per visualizzare le richieste, reqtools può essere configurata per mostrare i requester in una posizione specifica, può essere scaricata all'indirizzo seguente aminet
